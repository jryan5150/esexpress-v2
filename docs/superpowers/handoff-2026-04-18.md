### What the code does (from file headers)

- **pcs-auth.service.ts** — OAuth2 client-credentials grant against
  `POST {PCS_BASE_URL}/authorization/v1/tokens/oauth`. In-memory
  token cache keyed on client_id. Refresh when <10% of TTL remains.
  Singleflight so concurrent callers share one in-flight refresh.
  Persists snapshot to `pcs_sessions` with `sessionType='oauth'`.
- **pcs-rest.service.ts** — REST dispatch against `POST
  {PCS_BASE_URL}/dispatching/v1/load` with Bearer token from
  pcs-auth. Gated by `PCS_DISPATCH_ENABLED` env (**default false**
  = dry-run only). Scope intentionally limited to the dispatch push;
  status sync / webhooks / photo attachments stay on legacy until
  Phase 2 fully green-lit.
- Generated auth-client built against PCS Authorization API v1
  Swagger from 2026-03-06 (?). Load-client referenced in
  pcs-rest.service.ts but not in the untracked list — **verify it
  still exists and is checked in.**
