## The approved design (v5 Workbench)

**Single page** (`/workbench`) replaces Validation + BOL Queue + Dispatch Desk as daily workflow.

**Core concepts:**

- Every load is one row
- 4px color stripe (left edge) = whose turn (`assignments.current_handler_id` → user color)
- Phase badges = pickup vs delivery state (green/blue, from Stephanie's Feb 4 workflow session)
- Stage pill = where in the chain
- Contextual action = primary button, changes per stage

**Chain stages:** `uncertain | ready_to_build | building | entered | cleared`

**Stage transitions:**

- Uncertain → Ready to Build (Jess resolves all triggers)
- Ready to Build → Building (builder claims, Build+Duplicate N)
- Building → Entered (builder marks they typed in PCS)
- Entered → Cleared (Katie verifies)
- Cleared (terminal pre-settlement)

**6 uncertain triggers:** unassigned well · fuzzy match · BOL mismatch · weight mismatch · no photo 48h+ · rate missing

**Filters:** Uncertain · Ready to Build · Mine · Ready to Clear · Entered Today · All

**First-login onboarding:** 3 role variants (Jess / Steph+Scout / Katie), localStorage-gated, skippable, re-triggerable from user menu. Each walkthrough is 4-5 spotlighted steps.

**Data model changes:**

```sql
ALTER TABLE assignments
  ADD COLUMN handler_stage text NOT NULL DEFAULT 'uncertain',
  ADD COLUMN current_handler_id integer REFERENCES users(id),
  ADD COLUMN uncertain_reasons jsonb NOT NULL DEFAULT '[]',
  ADD COLUMN stage_changed_at timestamp with time zone DEFAULT now(),
  ADD COLUMN entered_on date;

ALTER TABLE loads
  ADD COLUMN pickup_state text,
  ADD COLUMN delivery_state text;
```

**Explicit non-goals in v5:** PCS REST push (blocked on OAuth), settlement UI (Jenny's role), rebuilding ingest/matcher, Vertex rule engine, JotForm→GCS pipeline (those are Phase 2-4 from the architecture plan).
