## 4. What the system found on its own

These are emergent findings the cross-check + matcher surfaced without operator prompting.

### Cross-check loop overnight

3 well-naming variants surfaced + resolved via single-click absorb:

- `Apache-Formentera-Wrangler` (34 loads) → "Liberty Apache Formentera Wrangler"
- `DNR - Chili 117X` (7 loads) → "Liberty Titan DNR Chili 117X"
- `Spectre-Crescent-SIMUL Briscoe Cochina` (13 loads) → "Liberty Spectre Crescent Briscoe Cochina"

Each was a 100% fuzzy match against an existing well. Aliases applied + 54 loads re-bound.

### Cross-check loop steady state

3 genuinely-novel discrepancies remain in the queue for human action:

- PCS load 357468 (status drift — Cancelled in PCS, was our 4/22 test push)
- Wells 1/2/3 (orphan destination, 888 loads — Sandbox loader, dispatcher decision)
- Victoria Anne West (orphan destination, 19 loads — same)

### Matcher case-sensitivity discovery

While diagnosing 1,417 stuck-pending JotForm submissions, sampling revealed PropX OCR extracted uppercase tickets (`C10698116`) while the loads table stored lowercase (`c10698116`). Matcher's strict equality missed them despite same-string-modulo-case.

Fix: `LOWER()` comparison. **Recovered 813 stuck submissions in one query.** This is the single biggest match-rate move of the day.

### Photo orientation finding

Driver-mobile photos carry EXIF orientation tags the workbench drawer didn't always respect. PCS push pipeline was already auto-rotating; the display proxy was not. Fixed with sharp.rotate() + 192 MB LRU cache so re-fetches are essentially free.

### PropX rate-limit discovery

Manual PropX sync at 17:18 UTC triggered ~80 rapid job-page fetches. PropX returned a wave of HTTP 401 ("Unauthorization Request") on roughly half — auth key was fine, this is their soft-throttle response. Rate limit halved (concurrency 4→2, intervalCap 10→4) + 401 added to retry policy.

In addition to the matching/data wins, these resiliency improvements landed:

All five external integrations now have explicit retry+breaker coverage. Up from 2 of 6 this morning.

## 6. What this proves

The system found and fixed itself:

- The 5.18% photo-coverage number was a missing-flag artifact, not a real coverage gap
- The 1,400+ stuck-pending was a case-sensitivity bug, not bad OCR
- The orphan-destination discrepancies were genuine surfacing of well-naming variance
- The PropX 401 wave was upstream throttling that needed our rate-limiter, not a key issue

Every bullet here came from the cross-check + JotForm-health diag layers — surfaces that didn't exist 48 hours ago. Each one is the kind of thing that previously sat invisible until a quarterly audit (or a chance reconciliation by Jenny / Jessica) caught it.

The bet for continuation: **the system finds these every day, automatically, and the curve compounds.**
